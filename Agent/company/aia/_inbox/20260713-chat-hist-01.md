## Turn 9: Grant Level Correction

### Sin's Correction:
> เดี๋ยวนะๆ อย่ามั่ว การ grant มี 2 lvl คือ grant ให้ group accout มองเห็น accces ได้
> กับ grant row lvl access คล้ายๆ rbac อ่ะ ไม่ได้มีแค่นั้นนี่

### Response Summary:

**Sin's correction accepted:**

**2 dimensions of access control:**

**Dimension 1: Object Privileges (Access Control)**
- Catalog level: USE CATALOG, CREATE SCHEMA, ALL PRIVILEGES
- Schema level: USE SCHEMA, CREATE TABLE, CREATE FUNCTION
- Table/View level: SELECT, MODIFY, ALL PRIVILEGES
- Function level: EXECUTE
- Analogy: key card entry

**Dimension 2: Data-Level Filtering (RLS/Column Mask)**
- Row Filter: function defined on table
- Column Mask: function on specific column
- Applied automatically per query
- Filter based on caller identity
- Analogy: internal visibility filter

**How they work together:**
```
Step 1: Privileges check → passed → query proceeds
Step 2: Row filter runs → returns filtered rows
Client sees: only their team's rows
```

**Both required:**
- Privileges alone → sees ALL rows (data leak)
- Row filter alone → permission denied

**RBAC analogy:**
```
Traditional DB RBAC:
- Roles (admin, editor, viewer)
- Row-level policies
- Column-level policies

Databricks UC RLS:
- Account groups (like roles)
- Row filter function (like row policies)
- Column mask function (like column policies)
= Same concept, Databricks-specific
```

**Complete setup example:**
```sql
-- Row filter function
CREATE OR REPLACE FUNCTION cost_platform.gold.team_row_filter(
    team_tag STRING
)
RETURN 
    is_account_group_member('de-team')
    OR is_account_group_member(concat('client-', lower(team_tag)));

-- Apply filter
ALTER TABLE cost_platform.gold.cost_wide
    SET ROW FILTER cost_platform.gold.team_row_filter ON (tag_team);

-- Grant privileges
GRANT USE CATALOG ON CATALOG cost_platform TO `client-underwriting`;
GRANT USE SCHEMA ON SCHEMA cost_platform.gold TO `client-underwriting`;
GRANT SELECT ON TABLE cost_platform.gold.cost_wide TO `client-underwriting`;
GRANT EXECUTE ON FUNCTION cost_platform.gold.team_row_filter
    TO `client-underwriting`;
```
