# 0012. Hard kill date 2026-08-31 — if unused by then, archive with the KB

- Status: Accepted
- Date: 2026-07-18
- Deciders: Wasin (+ portfolio-review 2026-07-18 §3.3, market-skeptic teeth)
- Tags: scope, lifecycle, discipline

## Context
The engine was built in ~1 day (2026-06-07), then dormant for 6 weeks — run only twice,
deterministic-only, the LLM path never run live. Critically it stopped 3 weeks *before*
the AIA job started, so the friction was the manual CLI, not lack of time. The proving-
ground value (b) — validating the agent-build workflow — has **already been harvested**
(visible in how Sin now runs the AIA sandbox). The only remaining value is (a) discipline
*if actually used*, which is currently 0.

## Options
- Keep indefinitely "just in case" — the portfolio's #7-unfunded-project trap: rewarding
  safe meta-artifacts over a market-facing action.
- Kill now — discards a finished, tiny, ~0-maintenance tool for no benefit.
- **Timebox with a hard trigger.**

## Decision
One weekend of work (backtest → automation → journal → deterministic gaps → Elliott/model
fix), then a **hard kill date of 2026-08-31**. The market-contact rule applies: the
market-facing action for crypto is *use it for real for one week*. If by 2026-08-31 it has
not been used on a real decision, **archive the repo together with its KB** — no guilt.
The agent-workflow lesson is a payoff already paid; parking costs ~0. Roadmap is frozen:
no dashboard, no predictive ML, no webhook leg.

## Consequences
- + Converts an open-ended "someday" into a falsifiable bet with a date.
- + Archiving is cheap and reversible (git history + KB retained).
- − Requires an honest check-in at end of August; the journal ([0011](0011-journal-required.md))
  is the evidence — empty journal on 2026-08-31 = archive.
- Review this ADR on 2026-08-31; flip Status to Superseded/Archived accordingly.
