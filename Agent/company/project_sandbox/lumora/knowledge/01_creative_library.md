# Creative Library — Framework v3 (canonical)

> The creative core of the **LUMORA** project.
> The framework is **catalog-agnostic** — LUMORA spans many catalogs (สายมู, อาหาร/สุขภาพ, ท่องเที่ยว, gadget, art…). สายมู (spiritual/mystical) is used throughout as **catalog #1** — the worked example — but every axis applies to any catalog.
> The library is a **divergent library**, grounded in established marketing theory.
> **Supersedes** the old 4-dimension model (Aesthetic × Subject × Format × Tone). That model is retired.

---

## The framework (2 levels)

The framework splits into **two levels**: things you fix once per account, and things you vary every post.

```
═══════════════════════════════════════════════════
  ACCOUNT-LEVEL  (fixed per channel, define ครั้งเดียว)
═══════════════════════════════════════════════════
  🎭 Voice / Archetype  — brand archetype + persona (Jung 12)
  👥 Audience Persona    — who is this account for
  🎯 Niche scope         — broad / narrow positioning

═══════════════════════════════════════════════════
  POST-LEVEL  (vary per post — the "Library" axes)
═══════════════════════════════════════════════════
  💎 Content (C1–C10)  — what to sell + topic (Pillar)
  🌍 Theme (open-ended) — world / setting / era
  📹 Media (M1–M12)     — format / production type

  Optional per-post tags:
  🎯 JTBD / Intent     — what job this post does for audience
  📊 Funnel (HHH)       — Hero / Hub / Hygiene
═══════════════════════════════════════════════════
```

### Why 2 levels?

- **Account-level = identity.** Voice, Audience, and Niche scope are slow-moving. ตั้งครั้งเดียว แล้ว lock — เปลี่ยนบ่อย = brand สับสน + algorithm สับสน. This is what makes one creator *feel like* one creator even when the visuals swing wildly.
- **Post-level = the divergent library.** Content × Theme × Media combine fresh every post. This matches Sin's creative thinking: diverge → keep diverging → combine selectively. ไม่ converge ก่อนเวลา.
- **Separation of concerns:** the wide visual range lives at post-level *without* destabilizing identity, because identity is pinned one level up.

## Account-level tags

Set these **once per account**. They are the fixed point everything else swings around.

### 🎭 Voice / Archetype (Jung 12)

Brand voice derives from a **Jungian archetype** (+ optional persona name). The 12 archetypes:

| Archetype | Archetype | Archetype | Archetype |
|---|---|---|---|
| Innocent | Sage | Explorer | Outlaw |
| Magician | Hero | Lover | Jester |
| Everyman | Caregiver | Ruler | Creator |

**Best fit สายมู (pick 1, blend 1-2):**

- **Magician** ⭐ — transformation, mystery, "make the unseen seen." Core มู energy.
- **Sage** — wisdom, explaining traditions, depth. ดึง "อยากดู intellectual" audience.
- **Explorer** ⭐ — *fellow explorer, not guru.* Safest framing for sacred content. ตรงกับ Sin มากสุด.
- **Jester** — comedy/skit pillar (C9). Self-deprecating only, never mocking believers.

> Sin's draft voice: a **curious traveler of spiritual aesthetics** — not a guru, not a comedian, both and neither. = Explorer × Magician × Jester blend.

### 👥 Audience Persona

A fictional character of the target audience. **How to write one:**

- **Demographics:** age range, location, occupation (e.g. "22-32, Bangkok working pro").
- **Psychographics:** beliefs, aesthetic taste, what they share.
- **The job they hire content for** (links to JTBD below).
- Keep it to **1-2 personas per account** — more = unfocused.

> Example: *"GenZ มู สาย aesthetic, 22-32, Bangkok working pro — believes a little, posts a lot, wants content that makes their feed look intentional."*

### 🎯 Niche scope

- **Broad** — wide theme (e.g. a whole catalog like สายมู overall), more aesthetics, bigger ceiling, slower algo positioning.
- **Narrow** — one pillar/aesthetic, fast algo lock, lower ceiling, easier monetization match.
- Sin's default: **broad theme, multi-aesthetic**, manage algo confusion via batching + per-account split (multi-account strategy handles narrowing later).

### 🔢 Channel count (how many channels to split into)

When deciding whether one audience should be **one channel or several**, use:

```
1 ≤ N ≤ S × A     — the V1/V2/V3 gates decide where N lands.
```

- **S** = subject / topic breadth · **A** = audience-segment (aesthetic) count.
- **Upper bound `S × A`** — the most channels you could ever justify (one per subject × segment cell).
- **Lower bound `1`** — a *single* channel can serve **all** S × A cells at once (Sin's own default is exactly this: one broad, multi-aesthetic channel). The bounds only frame the space; the arithmetic decides **nothing**.

> **Correction (2026-07-18):** the old lower bound `MIN(S,A) ≤ N` was **pseudo-math** — false. One channel can cover S subjects × A segments, so the true floor is `1`. Lumora's own broad-multi-aesthetic default violated the old floor. All the real decision power lives in the gates below, not the bounds.

**The V1/V2/V3 gates (this is what actually decides N):**

- **V1 — overlap threshold (~30%):** two candidate segments that overlap more than this should **merge** into one channel.
- **V2 — viable audience size (> ~10K):** a segment below this is **not worth its own channel** — fold it in.
- **V3 — positioning distinctness:** each candidate channel must pass *"is it meaningfully different?"* — if not, merge.

**Procedure:** list subjects (S) + segments (A) → apply **V1** (merge overlaps) → drop sub-**V2** segments → **V3**-test each survivor → the count that remains is N.

> ⚠️ **Over-splitting is the classic solo-creator failure mode.** For a one-person shop, every extra channel multiplies posting + maintenance load with **no shared-audience compounding** — you thin every channel and sustain none. **Default to N = 1.** Only split when a segment genuinely *fails V1* (too distinct to merge) **and** *passes V2* (big enough) **and** *passes V3* (distinct positioning). The formula's job is to make you justify each split against the gates — not to hand you a number.

## Optional per-post tags

Use these when you want sharper analytics. Skippable in Phase 1.

### 🎯 JTBD / Intent

What "job" does the audience *hire* this post for? **Tag format:**

> "When [situation], help me [outcome] so I can [reason]"

Example jobs:

- "Help me feel calm before work" → contemplative content
- "Help me know what to buy" → product recommendation
- "Help me feel cool about my beliefs" → aesthetic art
- "Help me laugh at myself" → comedy
- "Help me find sacred sites" → travel
- "Help me understand traditions" → educational
- "Help me look intellectual" → deep / cultural content
- "Help me decorate my space" → wallpaper / art
- "Help me explore new aesthetics" → art exploration

### 📊 Funnel stage (HHH)

Google's Hero / Hub / Hygiene production cadence:

- **Hero (5%)** — tentpole, viral push, brand awareness. High effort, rare.
- **Hub (35%)** — regular series, recurring format, audience retention. Medium effort.
- **Hygiene (60%)** — search / educational, evergreen, always-on. High volume.

## How to use (per post)

1. **Account context first.** The account already has Voice/Archetype + Audience Persona + Niche scope locked. Everything below stays in service of that identity.
2. **Pick the combo:** one **Content (C1-C10)** × one **Theme** × one **Media (M1-M12)**.
3. **(Optional) JTBD** — what job does this post do? Write it as "When… help me… so I can…".
4. **(Optional) HHH** — is this Hero, Hub, or Hygiene?
5. **Tag everything for analytics.** Store the full `C × Theme × M (+ JTBD + HHH)` as metadata alongside engagement → backend learns which combos perform per account.

**Iterate:** weekly — which `C × Theme × M` drove engagement? monthly — adjust pillar/theme mix. quarterly — revisit Audience Persona against actual followers.

> **Sin likes MULTIPLE aesthetics — do not converge.** When generating posts, *expand options across Theme and Media*; don't collapse the account into one look. The library stays wide; the **Voice** stays one. That single fixed point is what unifies the channel — not the visual style.
